# Sales One-Pager

Use this page for first executive conversations where time is limited and clarity matters.

## What SynapticFour Showcase is

SynapticFour Showcase is a customer-ready adoption framework that helps teams:

- start small with measurable technical value,
- integrate with existing delivery and pipeline investments,
- strengthen evidence quality for compliance-oriented reviews,
- scale toward user-facing intelligence workflows when ready.

## Why customers buy

- **Faster time-to-value:** first scoped proof can often run in weeks, not quarters.
- **Lower adoption risk:** modular rollout instead of big-bang deployment.
- **Stronger governance posture:** auditable technical evidence from real runs.
- **Cross-functional alignment:** one storyline for business, platform, and compliance teams.

## What makes this practical

- Reuses proven core components (`Ferrum`, `HELIOS`, `bioresearch-assistant`) instead of introducing parallel stacks.
- Supports existing pipeline ecosystems (for example Nextflow/Snakemake) via overlay patterns.
- Provides both executive and technical paths in one place, from question catalog to reproducible runbooks.

## Typical customer journey

1. Pick a path with [Which path fits me?](../for-customers/which-path.md)
2. Run a 90-minute discovery with the [Workshop guide](workshop-guide.md)
3. Execute a 2-week scoped proof and review results
4. Decide scale-up path (A/B/C/D combination)

## Outcomes to commit on early

- One business objective tied to one measurable technical outcome
- One defined proof scope (included/excluded systems)
- One stakeholder review checkpoint with clear go/no-go criteria

## Common executive questions

- "Can we start with only part of the platform?" -> Yes, start with Path A.
- "Do we need to replace current pipelines?" -> No, Path B overlays evidence on existing runs.
- "How does this help with EHDS-oriented expectations?" -> Path C creates repeatable technical evidence packages.
- "Can we support RAG on private data safely?" -> Path D adds controlled, source-grounded assistant workflows.
